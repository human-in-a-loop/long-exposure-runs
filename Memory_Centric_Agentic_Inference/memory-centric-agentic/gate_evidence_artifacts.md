# Gate Evidence Artifacts

M-EVIDART-1 turns the M-PRODREPLAY-1 manifest boundary into an operator production checklist. A manifest may declare a gate boolean, but every gate also needs a concrete evidence file path, a sha256 digest over the evidence bytes, matching identity fields, a valid measurement-time window, and the required upstream evidence digest links. The contract is generated by `scripts/build_gate_evidence_artifact_contract.py` into `data/gate_evidence_artifact_schema.csv`, `data/gate_evidence_required_fields.csv`, `data/gate_evidence_dependency_graph.csv`, and `data/gate_evidence_operator_checklist.csv`.

The required gates are root enrollment, attestation envelope, trust policy, intake custody, adapter conformance, timebase integrity, redaction integrity, evidence gatechain, uncertainty qualification, causal attribution, DC-001/DC-002 threshold replay, planner readiness boundary, and final handoff traceability. Each row names the manifest boolean, evidence path field, digest field, source system, identity binding, time-window rule, upstream dependency, and fail-closed absence reason. Fields such as `payload_digest`, `upstream_artifact_digests`, `identity_bindings`, and measurement windows must be independently produced by the gate evidence workflow rather than copied from the manifest.

`scripts/validate_gate_evidence_artifacts.py` validates only structure and linkage. It checks artifact presence, digest match, production evidence label, gate order, gate status, common identity continuity, measurement-window coverage, and upstream dependency links. It deliberately does not decide metric truth, threshold success, causal validity, production calibration, production readiness, or claim credit; those remain downstream responsibilities of the validated replay and claim-boundary chain.

The offline defect probes cover manifest-only gate booleans, missing evidence paths, nonexistent evidence files, digest mismatches, stale/out-of-window evidence, root identity mismatches, collector identity mismatches, topology/model/workload mismatches, downstream artifacts without upstream dependencies, and non-production evidence labels attempting promotion. The complete linked-artifact probe can set `evidence_artifact_complete=true` and `ready_for_production_target_replay=true`, but the boundary table still keeps `production_calibrated=false`, `production_ready=false`, `threshold_success=false`, `causal_validity_granted=false`, and `claim_credit_allowed=false`.

Reproduce:

```bash
python3 scripts/build_gate_evidence_artifact_contract.py
python3 scripts/validate_gate_evidence_artifacts.py
python3 scripts/plot_gate_evidence_artifacts.py
python3 tests/verify_gate_evidence_artifacts.py
python3 -m long_exposure.tools.promise_check <workspace>
python3 -m long_exposure.tools.org_check <workspace>
```
